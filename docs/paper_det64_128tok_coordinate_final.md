# A 64-D Coordinate-Conditioned Multimodal Aerosol Encoder for ARM SGP

## Abstract

This note documents the completed deterministic multimodal aerosol autoencoder trained on ARM SGP E13 aerosol and AOS meteorology data from 2016-11-29 through 2023-04-21. The model represents a 30-minute aerosol state with a 64-dimensional bottleneck latent vector. Each instrument is encoded as its own modality token, visible tokens are fused with a transformer, and coordinate-aware decoders retrieve size distributions as functions of diameter, CCN concentration as a function of supersaturation, and nephelometer scattering/backscattering as functions of observed humidification coordinates.

The final checkpoint used here is `last_checkpoint.pt` at epoch 790. The final validation selected cross-prediction loss is 0.2550 standardized MSE, with the best validation point in the final CCN refinement stage at epoch 750, 0.2424. Test leave-one-modality-out CCN prediction from all non-CCN inputs gives standardized MSE 0.2949 and skill 0.712 relative to a test-set mean baseline. The model is therefore already useful as a masked-instrument retrieval prototype, especially for CCN, nephelometer, SMPS, CPC, ACSM, and UHSAS. However, it remains an exploratory representation model rather than a validated physical retrieval product.

## Data

The run uses 112,030 half-hour rows after feature construction, covering 2016-11-29 to 2023-04-21 at ARM SGP E13. The split is a deterministic calendar-month hash split:

| Split | Rows |
|---|---:|
| Train | 78,573 |
| Validation | 16,031 |
| Test | 17,426 |

Input features are not raw netCDF variables directly. They are processed feature-window values generated by the project feature pipeline:

- Time base: 30-minute target rows.
- AOSMET: 1-minute features aggregated with mean and standard deviation.
- ACSM: 30-minute features.
- SMPS: 5-minute feature windows.
- APS, UHSAS, OPC, CPC, CCN, dry/wet nephelometer: 1-minute feature windows.
- Sizing distributions are interpolated to a common log-diameter grid from 3 to 30,000 nm.
- Concentrations and scattering responses use `log1p` transforms before normalization.
- Features are normalized using training-set mean and standard deviation.

The target feature mask excludes conditioning coordinates from target losses. For example, CCN supersaturation and nephelometer RH/T/P are used as decoder coordinates but are not themselves trained as aerosol target responses.

## Model

The completed run uses a deterministic structured transformer autoencoder. It does not use a VAE and does not use a special sizing-crosstalk subnet. Cross-instrument information exchange happens through normal transformer fusion.

Architecture summary:

| Component | Setting |
|---|---:|
| Model type | `structured_transformer_autoencoder` |
| Trainable parameters | 3,404,694 |
| Modality token width | 128 |
| Aerosol bottleneck | 64 |
| Encoder depth | 2 |
| Global transformer | 4 layers, 8 heads |
| Transformer feed-forward width | 2 x token width |
| Latent head hidden width | 256 |
| Decoder expansion | 4 layers, hidden width 256, output width 128 |
| Decoder depth | 3 |
| Size sequence encoder | diameter-aware transformer |
| Coordinate decoders | CCN, sizing, optical neph |

The network flow is:

1. Each instrument is converted into a 128-D modality token.
2. Missing individual features are represented with feature masks.
3. Hidden or unavailable instruments are removed from transformer attention with an input modality mask.
4. A learned latent query attends to visible modality tokens.
5. The latent query is compressed to a 64-D aerosol code.
6. The 64-D code is expanded back to a 128-D decoder state.
7. Separate decoder heads retrieve each measured target family.

## Coordinate-Conditioned Retrieval

The coordinate decoders separate aerosol state from query coordinate.

For CCN, the latent vector represents the aerosol state inferred from visible instruments. Supersaturation is then passed as a query coordinate to the CCN decoder:

`N_CCN = f(z_aerosol, supersaturation)`

This means the model can predict CCN concentration at supersaturations not hard-coded as separate output columns, as long as the queried supersaturation is within or near the training range.

Ambient RH is not the same thing as CCN counter supersaturation. RH can be part of the input state through AOSMET or nephelometer features. The CCN retrieval itself should be queried with supersaturation in percent, for example 0.2 for 0.2 percent supersaturation.

For sizing instruments, the same idea is used with log-diameter:

`dN/dlogDp = g(z_aerosol, log10(Dp))`

SMPS, APS, UHSAS, and OPC remain separate instrument-specific decoder heads because their physics and response functions differ.

For the nephelometer, scattering/backscattering is conditioned on the observed channel and humidification coordinates rather than predicted at arbitrary wavelength.

## Training Objective

The final run used a curriculum:

| Stage | Epochs | Purpose |
|---|---:|---|
| Instrument denoising token pretraining | 1-60 | Teach each instrument encoder to produce useful tokens from noisy/missing features. |
| Deterministic autoencode warmup | 61-100 | Reconstruct visible targets without cross-modal hiding. |
| Denoising autoencode | 101-140 | Add feature noise/dropout. |
| Mask-one mild | 141-360 | Hide one observed target modality and predict it from the others. |
| Mask-one denoise | 361-580 | Same masked task with input corruption. |
| Mask-one refine | 581-740 | Continue masked prediction without corruption. |
| CCN hidden refine | 741-790 | Focus final updates on CCN as the hidden target. |

The response loss is Huber loss, not plain MSE. Huber loss reduces the influence of rare extreme points while still behaving like squared error near zero. The objective also includes weighted aerosol closure and size spectral losses:

- Dry scattering from size/composition.
- Humidification response from dry/wet nephelometer.
- CCN activation ratio.
- Size spectral log-spectrum loss.
- Size moment loss.
- Size shape loss.

The training loss curve is therefore not directly comparable across stages. The validation cross-prediction MSE is the better indicator of retrieval behavior.

![Final loss diagnostics](../artifacts/temporal_gru_30min_20161129_20230421/run_det64_128tok_huber_mask1_coordinate_mps_reset_20260513/figures/final_loss_curve_clean_epoch790.png)

## Missing-Instrument Behavior

The model supports missing instruments through two mechanisms:

1. Feature masks remove missing feature values inside each modality.
2. The transformer receives an input modality mask, so hidden instruments cannot contribute attention evidence.

During mask-one training, one observed target modality is hidden, and the loss is applied to that hidden modality. This directly trains cross-instrument prediction. The current run does not train all possible visible-instrument subsets uniformly. Therefore, one-instrument retrieval is mechanically supported but not yet fully optimized.

## CCN Baseline Comparison

The traditional CCN baseline uses ACSM-derived bulk hygroscopicity and merged size distributions. The test-split command was:

```bash
python -m kappa_ccn_baseline.run_baseline \
  --prepared-arrays artifacts/temporal_gru_30min_20161129_20230421/features/prepared_arrays.npz \
  --split test \
  --fraction-basis mass \
  --output artifacts/kappa_ccn_baseline
```

The random-forest baseline is a non-neural data-driven comparison using the same broad information family as the kappa baseline: CCN supersaturation, ACSM chemistry, and merged size distribution. It trains a `RandomForestRegressor` on `log1p(N_CCN)`. Its inputs are the CCN supersaturation coordinate, ACSM organics/sulfate/ammonium/nitrate/chloride, and a merged SMPS/UHSAS/OPC/APS `dN/dlogDp` spectrum.

The matched comparison was run on the exact kappa baseline prediction rows, using 155,838 CCN observations from 5,852 test rows. This matters because the kappa and random-forest baselines require simultaneous usable ACSM, size spectra, and CCN observations, whereas the full neural leave-one-out aggregate has broader CCN coverage.

| Method | Inputs | Observations | Standardized log1p MSE | Skill vs matched mean | MAE cm^-3 | Bias cm^-3 | RMSE cm^-3 |
|---|---|---:|---:|---:|---:|---:|---:|
| Kappa-Kohler baseline | ACSM + merged size | 155,838 | 0.449 | 0.523 | 320.6 | +221.2 | 638.9 |
| Random forest | ACSM + merged size | 155,838 | 0.214 | 0.773 | 212.7 | -52.1 | 458.2 |
| Neural retrieval | all non-CCN instruments | 155,838 | 0.197 | 0.791 | 156.7 | -11.4 | 409.4 |
| Neural retrieval | ACSM + size only | 155,838 | 0.242 | 0.743 | 183.3 | +44.9 | 440.7 |
| Neural retrieval | nephelometer only | 152,898 | 0.414 | 0.548 | 319.8 | +72.4 | 567.0 |

On this matched test subset, both data-driven ACSM+size methods are much better than the kappa baseline. The full neural retrieval is best overall because it can also use CPC, nephelometer, and meteorology when available. The random forest is a strong baseline: it beats the final neural ACSM+size-only retrieval in standardized log1p MSE, but the neural ACSM+size-only retrieval has lower physical MAE and RMSE. This difference means the RF is doing slightly better on relative/log-space error, while the neural retrieval is better on absolute concentration error.

The nephelometer-only row uses only the `optical_neph` modality and hides CCN, ACSM, size instruments, CPC, and AOSMET. It has slightly fewer observations because 98 of the 5,852 matched test rows lack usable neph input. The result is scientifically useful: neph-only CCN prediction has positive skill, 0.548, so optical scattering and humidification response contain real CCN-relevant information. But it is much weaker than size+chemistry or full multimodal retrieval, so neph alone cannot replace direct size and composition information.

![Matched CCN model comparison including neph-only retrieval](../artifacts/ccn_model_comparison_final/final_ccn_model_comparison_with_neph_only.png)

The random-forest feature importances are useful because they expose a physically sensible shortcut learned by a flexible non-neural model. The dominant predictors are merged size bins near 100-200 nm and 50-100 nm, which are exactly the particle sizes expected to matter for activation at the observed supersaturations. ACSM chemistry contributes less total importance than size but is not negligible, with organic and nitrate among the strongest individual chemical predictors.

![Random forest CCN feature importance](../artifacts/ccn_model_comparison_final/random_forest_feature_importance_summary.png)

What this teaches us:

- Most of the predictable CCN signal is carried by the size distribution near the activation-relevant diameter range.
- Chemistry still helps, but in this RF setup it is secondary to size.
- A simple empirical model already beats the kappa closure by a large margin, so the neural model should be compared against RF, not only against kappa.
- The neural model's advantage is not merely nonlinear fitting of ACSM+size. Its strongest result comes from using additional modalities, especially CPC, optical scattering/humidification context, and meteorology.
- Nephelometer-only retrieval is informative but incomplete. It recovers part of the CCN signal through optical loading and humidification response, but without size distribution and chemistry it cannot match the stronger retrievals.

## CCN Inference Method

The new inference interface is implemented in:

- `aerosol_encoding/inference.py`
- `aerosol_encoding/infer_ccn.py`

Example command:

```bash
python -m aerosol_encoding.infer_ccn \
  --checkpoint artifacts/temporal_gru_30min_20161129_20230421/run_det64_128tok_huber_mask1_coordinate_mps_reset_20260513/last_checkpoint.pt \
  --input my_processed_feature_rows.csv \
  --output ccn_predictions.csv \
  --supersaturation 0.1 0.2 0.4 0.8
```

The input CSV must contain processed feature columns matching checkpoint feature names. To inspect the expected columns:

```bash
python -m aerosol_encoding.infer_ccn \
  --checkpoint artifacts/temporal_gru_30min_20161129_20230421/run_det64_128tok_huber_mask1_coordinate_mps_reset_20260513/last_checkpoint.pt \
  --write-template artifacts/temporal_gru_30min_20161129_20230421/run_det64_128tok_huber_mask1_coordinate_mps_reset_20260513/ccn_feature_template.csv
```

By default, `ccn_activation` is excluded from the input modalities so the retrieval cannot simply copy CCN. If CCN is intentionally being used as an input, pass `--use-ccn-input`.

The Python API is:

```python
from aerosol_encoding.inference import AerosolCCNRetriever

retriever = AerosolCCNRetriever("path/to/last_checkpoint.pt", device="cpu")
predictions = retriever.predict_ccn(
    features_dataframe,
    supersaturation_percent=[0.1, 0.2, 0.4, 0.8],
)
```

## Limitations

1. The final checkpoint is not necessarily the best checkpoint for every modality. Epoch 750 has the best selected validation metric in the final CCN stage, while epoch 790 is the true final state.
2. The model was trained mostly on mask-one prediction, not all-combinations masking. Single-instrument retrieval should be evaluated explicitly before deployment.
3. OPC coverage is sparse and validation OPC has no usable rows in the current split.
4. The input interface expects processed feature-window values. It is not yet a raw netCDF-to-retrieval pipeline.
5. The CCN decoder queries supersaturation, not ambient RH. RH is an input/context variable, not a substitute for supersaturation.
6. The model learns correlations in ambient observations. It does not prove causality or replace physical closure tests.

## Recommended Next Run

The next training setup should preserve this coordinate-conditioned architecture but add explicit missing-subset training:

1. Keep 128-D modality tokens and the 64-D bottleneck as the default baseline.
2. Add a scheduled all-combinations masking stage after mask-one training.
3. Add a single-visible-modality diagnostic stage without necessarily optimizing only for it.
4. Save both global-best and stage-best checkpoints with explicit names.
5. Evaluate CCN retrieval against a traditional size/composition/kappa baseline on matched test periods.
6. Add raw netCDF or feature-store inference utilities so the retrieval can start from actual instrument files rather than processed feature CSVs.
